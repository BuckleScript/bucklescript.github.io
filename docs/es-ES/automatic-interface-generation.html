Automatic Interface Generation

"Archivos de interfaz" (.mli, .rei archivos) son la "Descripción pública" de sus "archivos de aplicación correspondientes" (.ml, .re), expuesto como documentación y que contiene nada más que las declaraciones de tipo. Como este archivo es un módulo, un archivo de interfaz es esencialmente una firma de módulo.

Consejos y sugerencias

No tienes que escribir explícitamente un archivo de interfaz; por defecto; se inferirá uno desde el archivo de implementación (así como un tipo de módulo puede ser inferido cuando pasas el ratón sobre él) y todos los enlaces del archivo serán exportados. Recomendamos que, después de que termines las iteraciones en tu proyecto:

  • Explícitamente añade archivos de interfaz a los archivos hechos para ser públicos
  • Añade comentarios dockblock encima de cada enlace que sirva de documentación
  • Has algunos tipos abstractos, y simplemente no expongas cada enlace del archivo de interfaz

Algunos tipos se tienen que copiar pegar desde el archivo de implementación, lo que es tedioso. Por esto te permitimos generar automáticamente archivos de interfaz, los cuáles después puedes modificar como tú quieras.

Para el archivo sr/MyUtils.ml, ejecuta:

bsc lib/bs/src/MyUtils-MyProject.cmi

Dónde MyProject es el namespace de tu proyecto. Si no está activado, será sólo MyUtils.cmi). .cmi es el archivo OCaml/BuckleScript que contiene alguna información del tipo compilado.

El comando anterior arroja un boilerplate de interfaz .mli al stdout. Si estás usando Reason, conviértelo a la sintaxis .rei:

bsc -bs-re-out lib/bs/src/MyUtils-MyProject.cmi

si tu no tienes bsc habilitado globalmente, usa el proveído localmente en node_modules/bs-platform/lib/bsc.exe.

Nota: el modelo generado puede contener las cadenas "BS-externo" o "BuckleScript externo". Esto ocurre cuando usted ha utilizado @bs externos en el archivo de implementación. Es un error temporal, necesitas convertir manualmente estos "BS-EXTERNAL" de nuevo al correcto @bs externo por ahora. Corregiremos esto en un futuro.
