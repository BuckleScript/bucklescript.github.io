Conditional Compilation

This doesn't work with the Reason syntax right now, only the OCaml syntax.

A veces quieres escribir un código que trabaje con diferentes versiones de compiladores y librerías.

La gente solía usar los preprocesadores como preprocesador C para la familia de lenguajes C. La comunidad OCaml utiliza varios preprocesadores: cppo, ocp-pp, camlp4 IFDEF macros, optcomp y ppx optcomp.

En vez de un preprocesador, BuckleScript añade un nivel de lenguaje de compilación if estática. Es menos poderoso que otros preprocesadores ya que sólo es compatible con if (no con #define, #undefine, #include), pero hay varias ventajas.

  • Es pequeño (sólo ~500 lineas) y es altamente eficiente. Todo puede hacerse en un solo paso. Es fácil reconstruir el preprocesador en un archivo único, sin dependencias en librerías del compilador, que devolverlo a los compiladores viejos de OCaml.
  • Es puramente funcional y del tipo seguro, y coopera con herramientas de edición como Merlin.

Nota: La compilación condicional de BuckleScript no funciona aún con Reason.

Uso

lwt_unix.mli

type open_flag =
    Unix.open_flag =
  | O_RDONLY
  | O_WRONLY
  | O_RDWR
  | O_NONBLOCK
  | O_APPEND
  | O_CREAT
  | O_TRUNC
  | O_EXCL
  | O_NOCTTY
  | O_DSYNC
  | O_SYNC
  | O_RSYNC
#if OCAML_VERSION =~ ">=3.13" then
  | O_SHARE_DELETE
#end
#if OCAML_VERSION =~ ">=4.01" then
  | O_CLOEXEC
#end

No tienes que añadir nada a la construcción para tener estos funcionando. El compilador bsc ya los entiende.

Variables Integradas & Personalizadas

Ve la salida de bsc.exe -bs-list-conditionals:

> bsc.exe -bs-D CUSTOM_A="ghsigh" -bs-list-conditionals
OCAML_PATCH "BS"
BS_VERSION "1.2.1"
OS_TYPE "Unix"
BS true
CUSTOM_A "ghsigh"
WORD_SIZE 64
OCAML_VERSION "4.02.3+BS"
BIG_ENDIAN false

Añade tus variables personalizadas a la mezcla con -bs-D MY_VAR="bla":

> bsc.exe -bs-D MY_VAR="bla" -bs-list-conditionals
OCAML_PATCH "BS"
BS_VERSION "1.2.1"
OS_TYPE "Unix"
BS true
MY_VAR="bla"
...

Sintaxis Concreta

static-if
| HASH-IF-BOL conditional-expression THEN //
   tokens
(HASH-ELIF-BOL conditional-expression THEN) *
(ELSE-BOL tokens)?
HASH-END-BOL

conditional-expression
| conditional-expression && conditional-expression
| conditional-expression || conditional-expression
| atom-predicate

atom-predicate
| atom operator atom
| defined UIDENT
| undefined UIDENT

operator
| (= | < | > | <= | >= | =~ )

atom
| UIDENT | INT | STRING | FLOAT
  • IF-BOL significa que #IF debería estar en el principio de una línea.

Reglas de Tipeo

  • el tipeo de INT es int
  • el tipeo de STRING es string
  • el tipeo de FLOAT es float
  • los valores de UIDENT vienen ya sea de valores ya construidos (con los tipos documentados) o una variable de ambiente, si esta es literalmente true, false entonces es un bool, de otra manera si es analizable por int_of_string entonces no es de tipo int, de otra manera si es analizable por float_of_string entonces es un float, si no debería ser una cadena
  • Dentro de lhs operator rhs, lhs y rhs siempre está el mismo tipo y retorno del boolean. =~ es un operador de versión semántica que requiere que ambos lados sean una cadena.

Las reglas de evaluación son obvias. =~ respeta la versión semántica, por ejemplo, el motor de subrayado

semver Location.none "1.2.3" "~1.3.0" = false;;
semver Location.none "1.2.3" "^1.3.0" = true ;;
semver Location.none "1.2.3" ">1.3.0" = false ;;
semver Location.none "1.2.3" ">=1.3.0" = false ;;
semver Location.none "1.2.3" "<1.3.0" = true ;;
semver Location.none "1.2.3" "<=1.3.0" = true ;;
semver Location.none "1.2.3" "1.2.3" = true;;

Consejos & Sugerencias

Esta es una muy pequeña extensión de OCaml. Es compatible hacia atrás con OCaml excepto en el siguiente caso:

let f x =
  x
#elif //

#elif en el inicio de una linea es interpretado como un if estático. no hay problema con #if o #end, ya que ya son palabras clave.
