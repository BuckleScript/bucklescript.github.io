Difference from Native OCaml

Esto es particularmente importante cuando se porta una aplicación OCaml existente a JavaScript.

Tipos de Datos Personalizados

En OCaml, el FFI C permite al usuario definir un tipo de dato personalizado y personalizar el comportamiento caml_compare, caml_hash, etc. Esto no está permitido en nuestro backend (ya que no tenemos FFI C).

(In)equidad Física

En general, sólo usamos equidad física como una técnica de optimización; no confiamos en su exactitud, ya que es bien emparejado en el tiempo de ejecución.

Rango de Caracteres de Cadena

Actualmente, BuckleScript asume que el rango de caracteres es de 0-255. El usuario debería ser más cuidadoso cuando pase una cadena de JavaScript al lado de OCaml. Estamos trabajando en una solución para este problema.

Mapa Débil

El mapa débil de OCaml's no está disponible en BuckleScript. El puntero débil es reemplazado por un puntero estricto.

Enteros

OCaml tiene tipos int, int32, nativeint y int64.

  • Ambos int32 y int64 en BuckleScript tienen las mismas semánticas que en OCaml.
  • int en BuckleScript es lo mismo que int32 mientras que en OCaml es dependiente de la plataforma.
  • nativeint es tratado como un float de JavaScript, excepto para división. Por ejemplo, Nativeint.div a b será traducido a a / b | 0.

Nota: Nativeint.shift_right_logical x 0 es diferente de Int32.shift_right_local x 0. La primera es literalmente traducida a x >>> 0 (traducido a un int sin firma), mientras que el segundo es x | 0.

Printf.printf

La implementación Printf.print en BuckleScript requiere una nueva línea (\n) para activar la impresión. Este comportamiento no es consiste con el comportamiento amortiguado del OCaml nativo. El único problema potencial que podemos preveer es que si el programa termina sin ningún caracter de nueva linea, el texto nunca será impreso.

Módulo Obj

Hacemos todo lo posible para mimetizar al compilador nativo, pero no tenemos garantías y hay algunas diferencias.

Algoritmo Hashtbl Hash

BuckleScript usa el mismo algoritmo que OCaml nativo, pero la salida es diferente debido a la representación en el tiempo de ejecución de int/int64/int32 y float.

Módulo Marshall

Aún no está soportado.

Str Module

Not supported as it is implemented in C, which is not portable to BuckleScript. Use the Js.String module instead which has bindings to the JavaScript String class.

Sys.argv, Sys.max_array_length, Sys.max_string_length

Command line arguments are always empty. This might be fixed in the future. Sys.max_array_length and Sys.max_string_length will be the same as max_int, but it might be respected.

Unsupported IO Primitives

Because of the JS environment limitation, Pervasives.stdin is not supported but both Pervasives.stdout and Pervasives.stderr are.
