Concepts Overview

開始接下來的章節之前，幾個有用的東西需要知道：

OCaml

這是 BuckleScript 的後盾。 BuckleScript 是 OCaml 的分叉，明確地版本是 v4.02.3（即將升級！）。 此文件假設你有 OCaml 的基本知識；你可以透過 Real World OCaml 學習 OCaml，或者，如果你正在學習 Reason，從 Reason 網站開始。

此文件主要包含額外加入讓 OCaml 能夠與 JS 良好交互作用的內容。

Reason

OCaml 的替代語法，對 JS 程式設計師比較熟悉，減少一些學習成本。 Reason 與 OCaml 一起運作，除了語法不同外，其他大多數東西是共用。 這表示 Reason 也能與 BuckleScript 一起運作。 有些頭等支援供 Reason 使用。

OPAM

這是 OCaml 的官方套件管理工具。由於 BuckleScript 使用 NPM 和 Yarn，初學者並不需要 OPAM。我們將在進階工作流程中提到它。

External/Interop/FFI

這些單字都跟 BuckleScript 有關 <-> JavaScript。

External：在 BuckleScript 情境中，一種主要的方式用來使用 JS 的值。

Interop："互用性（interoperability）" 的縮寫。

FFI：外來函式介面（Foreign Function Interface）。綜合詞彙像是 "external"，"wrapper" 及 "interop"。基本上表示呼叫一個來自其他語言的值。偏好前兩個詞彙。
