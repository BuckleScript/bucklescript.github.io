What & Why

O que é o BuckleScript?

BuckleScript não é uma nova linguagem. Ele simplesmente usa OCaml, uma linguagem rápida, pragmática e tipada, e a compila para código JavaScript limpo, legível e de alto desempenho. Isso permite que os usuários escrevam em uma linguagem bastante usada na indústria, ao mesmo tempo usando as ferramentas familiares existentes em JavaScript (npm/yarn, bundlers, minifiers, devtools, etc.) e com acesso ao vasto ecossistema de JavaScript e OCaml.

Graças a sua grande interoperabilidade com JS, BuckleScript pode ser vista como uma linguagem "gradualmente" tipada, onde você começa a adotá-la incrementalmente em seu código arquivo por arquivo, mantendo os benefícios de um sistema de tipagem 100% seguro e confiável, além de funcionalidades extra da linguagem.

BuckleScript possui suporte embutido para Reason, um projeto parceiro nosso, proporcionando uma experiência ainda mais familiar aos desenvolvedores JS existentes.

Porquê Compilar para JavaScript?

O ecossistema e comunidade de JavaScript são muito vibrantes e ativas. A linguagem em si é incomparável em termos de distribuição e adoção. Compilar para JavaScript é portanto uma escolha pragmática (e às vezes, a única opção).

Além disso, as VMs de JavaScript também receberam muitos anos de desenvolvimento e ajuste fino e ficam mais rápidas o tempo todo.

Porquê BuckleScript?

"Por que não usar só JavaScript então?"

Apesar de suas vantagens óbvias, JavaScript apresenta alguns inconvenientes. Eis como o BuckleScript os resolve.

Confiabilidade

O sistema de tipagem do BuckleScript em si oferece três vantagens principais para tornar seus programas mais confiáveis:

  • Sistema de tipagem seguro. "Seguro" aqui significa que os tipos garantem que eles são o que são, não apenas 90% do tempo. Uma vez que um projeto de BuckleScript compila, não há nenhum error de tipagem em tempo de execução. *
  • Inferência de tipo forte. Quase a totalidade da linguagem pode ser inferida. Você não tem que tediosamente escrever todos os tipos de manualmente. Sinta-se à vontade de incluir tipos para legibilidade!
  • Funcionalidades de typos expressivas. Com funcionalidades bem pensadas como variantes, módulos e mesmo um sistema de objetos opcional, os tipos te guiam pelo processo de iteração e não te impedem de expressar o que você precisa.

A combinação destes três destaques, junto com outros recursos do sistema de tipagem, abre novas possibilidades de workflow e é algo que as soluções existentes de tipagem graduais para JavaScript não oferecem.

* As exceções são alguns conectores de interoperabilidade opcionais.

Eliminação de código morto de alta qualidade

O ecossistema de JavaScript é tem muitas dependências e frequentemente muitas delas. O produto final inevitavelmente contém uma enorme quantidade de código, muito do qual o projeto não usa realmente. Essas regiões de código morto impactam o carregamento, análise e interpretação velocidade. O BuckleScript fornece eliminação de código morto poderoso em todos os níveis:

  • Eliminação de código a nível de módulo e função é facilitada pelo sistema de tipagem bem desenvolvido e análise de pureza.
  • A nível global, a BuckleScript gera código que é naturalmente amigável para eliminação de código morto feita por ferramentas de empacotamento como Rollup e Closure Compiler , após sua própria etapa de eliminação sofisticado.
  • O mesmo se aplica para minúsculo runtime do BuckleScript (que é escrito em BuckleScript mesmo).

Otimizações de tempo de compilação

JavaScript é uma linguagem dinâmica; ela sofre perda de desempenho atingido quando o engine do navegador otimiza o código em tempo de execução. Enquanto alguns engines contornam o problema em certa medida via caching, isto não está sempre disponível nem é previsível. A falta de um sistema de tipo forte embutido também limita a quantidade de otimizações possíveis. BuckleScript, por outro lado, é capaz de fornecer muitas otimizações durante o tempo de compilação, muito antes que o código seja enviado para o navegador. Um adágio generalizado para escrever código JavaScript rápido é escrever como se houvesse um sistema do tipagem (a fim de permitir boas heurísticas de otimização dos engines de JS); BuckleScript te oferece uma sistema de tipagem de verdade e gera código que é amigável para otimizações por padrão.

Roda em Native Também!

BuckleScript is simplesmente OCaml (exceto a interoperabilidade com JavaScript). OCaml pode compilar para código de máquina também (i. e. assembly puro e rápido). Quando você não está compilando para JavaScript, você pode reusar o mesmo código para servidores, desktops, sistemas operacionais de celular, etc. Isso permite desempenho ainda melhor. Por examplo, a aplicação tradicional node.js inicializa em >100ms, enquanto OCaml compilad para código de máquina inicializa em mais ou menos 2ms.

* De fato, ele até roda em microcontroladores com apenas alguns KBs de memória.

Como o BuckleScript é differente?

Hoje em dia, há muitas linguagens de programação que compilam para JavaScript. BuckleScript se diferencia das alternativas em alguns pontos cruciais.

Ciclo de iteração rápido

O tempo de construção do BuckleScrip é uma ou duas ordens de magnitude mais rápido que as alternativas. Em modo watcher, o sistema de construção geralmente termina antes que você troque a tela do editor para o terminal (dois dígitos de mili-segundos). Um ciclo de iteração rápido reduz a necessidade de se manter estado mental por muito tempo; o que por sua vez permite continuar no fluxo por mais tempo e com mais frequencia.

Resultado legível & boa interoperabilidade

Código JavaScript ilegível gerado a partir de outras linguagens compiladas para Js faz com que seja, em termos práticos:

  • Difícil de depurar (stack trace críptico, nomes de variáveis retorcidos)
  • Difícil de se aprender (mapeamento não-trivial de conceitos de uma linguagem para a outra)
  • Difícil de medir desempenho (difícil dizer qual o desempenho de tempo de execução)
  • Difícil de integrar com código existente de JS escrito à mão

A saída JS do BuckleScript é bastante legível. Isto é especialmente importante durante aprendizagem, onde os usuários podem querer entender como o código é compilado e fazer auditoria de bugs.

Esta característica, combinada com um sistema completo de interoperabilidade com JS, permite que código BuckleScript seja inserido em uma base de código de JavaScript existente quase despercebido.

Código JS minúsculo

Um programa Olá mundo em BuckleScript gera 20 bytes de código JS. Além do mais, as partes da biblioteca padrão que você precisa só são incluídas quando necessário.

Preservação da estrutura do código

O BuckleScript mapeia cada arquivo OCaml/Reason para um arquivo JavaScript. Isso facilita a integração de ferramentas existentes como bundlers e frameworks de testes. Você pode até mesmo começar a escrever um único arquivo sem ter que mudar o seu setup de build. A estrutura de código de cada arquivo também é preservada aproximadamente.
